# codex-account-switcher v0.1.19

## Summary
This patch prepares the next publishable release and makes fresh npm releases show up in codex-auth update prompts much sooner after publish.

## What's changed
- Updated package metadata from `0.1.18` to `0.1.19`.
- Shortened the cache TTL for cached "up to date" npm version checks so freshly published versions stop staying invisible for hours.
- Kept the longer cache window for already-known update-available results to avoid unnecessary npm checks.

## Verification
- `npm test`
- `npm pack --dry-run`

## Upgrade notes
- No breaking CLI changes.
- Users should see new-version prompts soon after a release instead of waiting for the old six-hour negative cache to expire.

## Publish checklist
1. Review package metadata and release notes.
2. `npm test`
3. `npm pack --dry-run`
4. `npm publish --access public`
5. Create GitHub release `v0.1.19` from `releases/v0.1.19.md`
