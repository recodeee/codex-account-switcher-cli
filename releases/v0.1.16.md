# codex-account-switcher v0.1.16

## Summary
This patch reduces steady-state `codex-auth` startup latency and prepares the next publishable npm version.

## What's changed
- Cached interactive npm update checks used by bare `codex-auth` and interactive `codex-auth list`, so they stop hitting the registry on every startup.
- Kept explicit `codex-auth self-update` on the live npm lookup path.
- Added a session-scoped fast path that skips external auth reconciliation when the current terminal already saw the same unchanged regular `~/.codex/auth.json`.
- Added regression coverage for cached update checks and unchanged-auth sync short-circuiting.

## Verification
- `npm --prefix codex-account-switcher test --silent`
- `npm --prefix codex-account-switcher pack --dry-run`

## Upgrade notes
- No breaking CLI changes.
- Interactive update prompts may reuse cached npm metadata for several hours before refreshing.
