# Supabase workspace (Medusa backend)

This folder keeps Medusa-adjacent SQL for the separate commerce service.

## What belongs here

- Additive SQL owned by the commerce layer (`commerce` schema)
- Seed metadata required by the Medusa service startup
- Supabase project config used for `supabase db push`

## What does **not** belong here

- Medusa core table definitions (`public.*`) generated by Medusa migrations
- codex-lb Python proxy/dashboard schemas

## Workflow

1. `cd backend/apps/backend`
2. `supabase login`
3. `supabase link --project-ref <project-ref>`
4. Review `supabase/migrations/*.sql`
5. `supabase db push`

The Medusa runtime uses `SUPABASE_DB_URL` (or `DATABASE_URL` fallback) from `.env`.
