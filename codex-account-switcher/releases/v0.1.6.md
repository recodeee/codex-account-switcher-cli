# codex-account-switcher v0.1.6

## Summary
This patch improves login snapshot reliability, account-name uniqueness, and auth metadata detection.

## What's changed
- `login` command now supports optional name inference:
  - `codex-auth login [name]`
  - `codex-auth login [name] --device-auth`
  - `codex-auth login [name] --force`
- Added a post-login snapshot readiness wait so the command only saves after refreshed auth is detectable.
- Improved auth metadata parsing fallbacks:
  - email fallback from JWT payload or root
  - account ID fallback from `account_id`, `default_account_id`, claim, or root
  - user ID fallback from claim fields or JWT `sub`
  - plan fallback from claim/payload/root
- Improved unique-name inference to distinguish same-email but different account identities.
- Improved account matching so selectors can match by `accountId` and `userId` in addition to name/email.
- Added/expanded tests for parser fallbacks and identity-safe save behavior.
